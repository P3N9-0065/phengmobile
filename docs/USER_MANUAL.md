## 1. ການເຂົ້າສູ່ລະບົບ

1. ເປີດ URL ຂອງລະບົບ → ໄປໜ້າ `/login`
2. ໃສ່ອີເມວ + ລະຫັດຜ່ານ ຫຼື ກົດ **Sign in with Google**
3. ຖ້າເປັນບັນຊີໃໝ່ທີ່ຍັງບໍ່ໄດ້ມອບສິດ → ຈະເຫັນຂໍ້ຄວາມ "ບັນຊີຍັງບໍ່ໄດ້ຮັບສິດ" → ໃຫ້ admin ໄປ **ຜູ້ໃຊ້ງານ** ມອບ role ໃຫ້ກ່ອນ

**ອອກຈາກລະບົບ:** ກົດ **ອອກຈາກລະບົບ** ທີ່ມຸມຊ້າຍລຸ່ມ

---

## 2. ສິດການນຳໃຊ້

| Role | ສິດເຂົ້າເຖິງ |
|---|---|
| **admin** | ທຸກໜ້າ + ຕັ້ງຄ່າ + ຈັດການຜູ້ໃຊ້ + ລາຍງານ |
| **cashier** | Dashboard, POS, ໃບສ້ອມ, ບິນຂາຍ, ສັ່ງຊື້ອອນລາຍ, ລູກຄ້າ, ສະໝັກບັນຊີ |
| **technician** | Dashboard, ໃບສ້ອມ |
| **warehouse** | Dashboard, ສະຕັອກ, ຜູ້ສະໜອງ |

> ຄົນໜຶ່ງສາມາດມີຫຼາຍ role ໄດ້

## 3. ໜ້າລວມ (Dashboard)

ສະຫຼຸບສະຖານະທຸລະກິດປະຈຳວັນ:
- ຍອດຂາຍວັນນີ້ / ເດືອນນີ້
- ຈຳນວນບິນ
- ໃບສ້ອມທີ່ກຳລັງດຳເນີນການ
- ສິນຄ້າເຫຼືອນ້ອຍ (low stock)
- ການແຈ້ງເຕືອນຕ່າງໆ

---

## 4. ຂາຍໜ້າຮ້ານ (POS)

**ການຂາຍສິນຄ້າ:**
1. ໄປ **ຂາຍ (POS)**
2. **ສະແກນບາໂຄດ** ດ້ວຍປືນຍິງ ຫຼື ກ້ອງມືຖື ຫຼື ພິມຄົ້ນຫາຊື່/SKU
3. ປັບຈຳນວນໃນຕະກຣ້າ
4. ເລືອກ/ສ້າງລູກຄ້າ (optional — ຈຳເປັນຖ້າຢາກສະສົມຄະແນນ)
5. ເລືອກວິທີຊຳລະ: ເງິນສົດ / ໂອນ / QR
6. ກົດ **ບັນທຶກບິນ** → ພິມໃບບິນ

**ຄຸນສົມບັດເພີ່ມເຕີມ:**
- ສ່ວນຫຼຸດແບບ % ຫຼື ກີບ
- ໃຊ້ຄະແນນສະສົມຫັກໄດ້ (loyalty points)
- ຄືນສິນຄ້າ — ໄປ **ບິນຂາຍ** → ເລືອກບິນ → **ຄືນ**

---

## 5. ໃບສ້ອມ (Repairs)

**ສ້າງໃບສ້ອມໃໝ່:**
1. **ໃບສ້ອມ** → **+ ໃບສ້ອມໃໝ່**
2. ຂໍ້ມູນລູກຄ້າ (ເລືອກ/ສ້າງ)
3. ຂໍ້ມູນເຄື່ອງ: ຍີ່ຫໍ້, ລຸ້ນ, IMEI, ສີ, ລະຫັດປົດລັອກ
4. ອາການເສຍ, ຄ່າປະເມີນ
5. ຖ່າຍຮູບສະພາບເຄື່ອງ (ກ່ອນສ້ອມ)
6. ກົດ **ສ້າງໃບສ້ອມ** → ພິມໃບຮັບ

**ສະຖານະ:**
- `received` ຮັບເຄື່ອງແລ້ວ
- `diagnosing` ກຳລັງກວດ
- `in_repair` ກຳລັງສ້ອມ
- `awaiting_parts` ລໍອາໄຫຼ່
- `ready` ສ້ອມສຳເລັດ — ແຈ້ງລູກຄ້າມາຮັບ
- `delivered` ສົ່ງມອບແລ້ວ
- `cancelled` ຍົກເລີກ

**ການຕິດຕາມ:** ລູກຄ້າເຂົ້າ `/track/<code>` ເພື່ອເບິ່ງສະຖານະ

---

## 6. ສະຕັອກ (Inventory)

**ເພີ່ມສິນຄ້າ:**
1. **ສະຕັອກ** → **+ ສິນຄ້າ**
2. ໃສ່: ຊື່, SKU, ບາໂຄດ, ໝວດ, ລາຄາທຶນ, ລາຄາຂາຍ, ຈຳນວນ
3. **ຖ່າຍຮູບ** (ສຳຄັນສຳລັບໜ້າຮ້ານອອນລາຍ)
4. ຕັ້ງ **is_featured** = true ຖ້າຢາກໃຫ້ສະແດງໃນຮ້ານອອນລາຍ

**ການເຄື່ອນໄຫວສະຕັອກ:**
- POS ຂາຍ → ຫັກສະຕັອກ ອັດຕະໂນມັດ
- ໃບສ້ອມໃຊ້ອາໄຫຼ່ → ຫັກສະຕັອກ
- **ປັບສະຕັອກ** ດ້ວຍມື → ມີບັນທຶກໃນ stock_movements

**ສະແກນບາໂຄດ:** ໃຊ້ປຸ່ມ scan ເພື່ອຄົ້ນຫາສິນຄ້າດ່ວນ

---

## 7. ລູກຄ້າ (Customers)

- ເບິ່ງລາຍຊື່ລູກຄ້າທັງໝົດ
- ປະຫວັດການຊື້, ການສ້ອມ
- ຄະແນນສະສົມ (loyalty points)
- ຍອດໃຊ້ຈ່າຍລວມ
- ຄົ້ນຫາດ້ວຍຊື່ / ເບີໂທ

---

## 8. ບິນຂາຍ (Sales)

- ລາຍການບິນທັງໝົດ
- Filter ຕາມວັນທີ, ລູກຄ້າ, ວິທີຊຳລະ
- ກົດເຂົ້າເບິ່ງລາຍລະອຽດ → ພິມຊ້ຳ ຫຼື ຄືນສິນຄ້າ

---

## 9. ສັ່ງຊື້ອອນລາຍ (Orders)

ບ່ອນຈັດການອໍເດີຈາກໜ້າຮ້ານອອນລາຍ:

**ສະຖານະອໍເດີ:**
- `pending_payment` ລໍລູກຄ້າອັບສລິບ
- `payment_uploaded` ມີສລິບແລ້ວ ລໍກວດ
- `payment_verified` ກວດສລິບຜ່ານ → ກຽມສົ່ງ
- `shipped` ສົ່ງແລ້ວ
- `delivered` ລູກຄ້າຮັບແລ້ວ
- `cancelled` ຍົກເລີກ

**ການກວດສລິບ:**
- ລະບົບ OCR ກວດອັດຕະໂນມັດເມື່ອລູກຄ້າອັບໂຫລດ
- ກວດສລິບຊ້ຳ (duplicate detection) ກັນຄົນໃຊ້ສລິບເກົ່າ
- Admin ສາມາດກົດ **ກວດໃໝ່** ດ້ວຍມືໄດ້

**ຂັ້ນຕອນຈັດການ:**
1. ຮັບແຈ້ງເຕືອນອໍເດີໃໝ່
2. ກວດສລິບ → ຖ້າຖືກຕ້ອງ ກົດ **ຢືນຢັນຊຳລະ**
3. ກຽມສິນຄ້າ → ກົດ **ສົ່ງແລ້ວ** + ໃສ່ເລກພັດສະດຸ
4. ລູກຄ້າຍືນຍັນຮັບ → ສະຖານະເປັນ `delivered`

## 10. ຜູ້ສະໜອງ (Suppliers)

- ບັນທຶກຂໍ້ມູນຜູ້ສະໜອງ
- ປະຫວັດການສັ່ງຊື້
- ສ້າງ Purchase Order (PO)

---

## 11. ສະໝັກບັນຊີ (Account Signups)

ສຳລັບລົງທະບຽນບໍລິການເຄືອຂ່າຍ (eSIM, ເບີໃໝ່ ຯລຯ):
1. ສ້າງໃບສະໝັກ → ໃສ່ຂໍ້ມູນລູກຄ້າ + ບໍລິການ
2. ອັບໂຫລດສລິບ/ເອກະສານ
3. ສະຖານະ: pending → approved → completed

---

## 12. ລາຍງານ (Reports) — admin only

- ຍອດຂາຍຕາມຊ່ວງເວລາ
- ສິນຄ້າຂາຍດີ
- ກຳໄລຂາດທຶນ
- Export ເປັນ CSV / Excel

---

## 13. ຜູ້ໃຊ້ງານ (Users) — admin only

**ມອບສິດໃຫ້ພະນັກງານ:**
1. ພະນັກງານ sign up ດ້ວຍ Google/ອີເມວກ່ອນ
2. Admin ໄປ **ຜູ້ໃຊ້ງານ** → ຫາຊື່ນັ້ນ
3. ຕິກ role ທີ່ຕ້ອງການ (admin / cashier / technician / warehouse)
4. ບັນທຶກ

**ຖອນສິດ:** ເອົາ role ອອກ → ຄົນນັ້ນຈະເຂົ້າລະບົບບໍ່ໄດ້ອີກ

---

## 14. ຕັ້ງຄ່າ (Settings) — admin only

- **ຊື່ຮ້ານ** (ລາວ + ອັງກິດ), ໂລໂກ້, ທີ່ຢູ່, ເບີໂທ
- **ບັນຊີທະນາຄານ** (ສຳລັບໜ້າຮ້ານອອນລາຍສະແດງ)
- **ຄ່າຂົນສົ່ງ** (shipping rates ຕາມເຂດ)
- **ຄະແນນສະສົມ** (loyalty rate, ມູນຄ່າແລກ)
- **ນະໂຍບາຍຄືນສິນຄ້າ** (return policy)
- **VAT** (ຖ້າມີ)

---

## 15. ໜ້າຮ້ານອອນລາຍ (Shop)

ໜ້າສຳລັບລູກຄ້າທົ່ວໄປ (ບໍ່ຕ້ອງ login):
- `/shop` — ເບິ່ງສິນຄ້າ, ໃສ່ຕະກຣ້າ, checkout
- `/track-order/<code>` — ຕິດຕາມສະຖານະອໍເດີ
- `/track/<code>` — ຕິດຕາມໃບສ້ອມ

**ຂັ້ນຕອນຂອງລູກຄ້າ:**
1. ເລືອກສິນຄ້າ → ໃສ່ຕະກຣ້າ
2. ກອກຊື່, ເບີ, ທີ່ຢູ່ສົ່ງ
3. ໂອນເງິນຕາມບັນຊີທີ່ສະແດງ
4. ອັບໂຫລດສລິບ → ໄດ້ລະຫັດອໍເດີ (ເຊັ່ນ `SO260611XXXX`)
5. ໃຊ້ລະຫັດເຂົ້າຕິດຕາມສະຖານະ

---

## 16. ການແກ້ໄຂບັນຫາທົ່ວໄປ

| ບັນຫາ | ວິທີແກ້ |
|---|---|
| Login ບໍ່ໄດ້ "Unsupported provider" | Admin ຕ້ອງເປີດ Google provider ໃນ backend |
| "ບັນຊີຍັງບໍ່ໄດ້ຮັບສິດ" | Admin ໄປ **ຜູ້ໃຊ້ງານ** ມອບ role |
| ສະຕັອກບໍ່ກົງ | ໄປ **ສະຕັອກ** → ປັບດ້ວຍມື + ໃສ່ເຫດຜົນ |
| ສລິບກວດບໍ່ຜ່ານ | ກວດດ້ວຍຕາ → ກົດ **ຢືນຢັນດ້ວຍມື** |
| ສິນຄ້າບໍ່ສະແດງໃນຮ້ານ | ກວດ `is_featured = true` |
| ບາໂຄດສະແກນບໍ່ໄດ້ | ໃຊ້ກ້ອງມືຖືແທນ ຫຼື ພິມລະຫັດດ້ວຍມື |
| ພິມບິນບໍ່ອອກ | ກວດການເຊື່ອມຕໍ່ printer / ໃຊ້ປຸ່ມພິມຊ້ຳ |
